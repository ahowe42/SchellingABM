Schelling's Model of Segregation

In the 1970's, economist Thomas Schelling started investigating the prevalence of residential segregation along racial and socio-economic lines. It was thought that segregation was so prevalent because certain classes of people have strong preferences to live among others like them. Schelling developed a simple game theory model with agents representing people living in neighborhoods.

Each agent has a similarity preference (SP) which dictates how many neighbors need to be similar for the agent to be happy. In the grid where each locus represents a home where an agent can live, the "neighborhood" of the agent includes the grid locations within distance 1. An SP of 8 indicates that an agent would only be happy if completely surrounded by similar agents. At the other extreme, SP=0 would indicate that an agent didn't care about the neighborhoods similarity.

When Schelling developed the model concept, computers were not as widespread as now, so he drew out a grid on paper, used coins for agents, and manually kept track of the results. His results suggested that large-scale segregation occur even when people only have weak preferences for similar neighbors.

Complex Adaptive Systems & Emergence

The power of the agent-based models such as this is in the complexity and patterns that arises from simplicity. In this case, each agent evaluates a simple relation for happiness: happy=[n(neighbors similar)≥SP]. If an agent is unhappy, it follows the simple rule of making a move to a randomly-selected unoccupied location on the grid. The process of evaluating happiness then possibly moving is iterated over all agents in sequence until some criteria is met. Even with low SPs, marked patterns of segregation arise as the model is iterated. The pattern of segregation can be called emergent behavior.

My Implementation

I have developed this model here in Python as a very simple project by which to get familiar with GITHUB (besides the fact that it's cool!). Agents are coded as objects, which is probably the most logical and flexible way to do that. What I've done that might be less typical is encoded each agent with a Race attribute, which is also a separate object. While each Person (the agent) object can have their own SP, the SP is currently an attribute of the Race object. This structure can easily be extended to allow each agent to have multiple characteristics along which similarity preference can be measured (age, education, etc…).

Sample Model Output

As a simple demonstration, we show here visual results from running the model for a pre-specified maximum of 500 iterations, using a 50×50 grid with 2,375 agents (5% unoccupied space). There are 3 races with SPs of 30%, 60%, and 20%. Their are 625, 1000, and 750 agents of each respective type, so the weighted average SP is 0.39. Using these numbers, the grid was randomly populated, and shown on the left pane of Figure 1. Agents are marked with a ° if they are happy, and × otherwise. Unfilled circles are empty locations. With the random initialization, approximately 42% of the agents were happy. The right pane of Figure 1 shows the rsulting pattern of segregation after 500 iterations, with about 85% of the agents happy. Note the marked patterns of segregation that have emerged as a result of the iterative application of the simple rules.

Figure 2 tells us some by-iteration information about the models evolution. In the left pane, we see that the number of happy agents converged very quickly to near 85% in fact, it had converged by iteration 20, and just made small corrections for the remaining 480 iterations. The right pane shows how many agents moved to new locations in each iteration. This metric similarly converged quickly to around 470 movers in each iteration.
